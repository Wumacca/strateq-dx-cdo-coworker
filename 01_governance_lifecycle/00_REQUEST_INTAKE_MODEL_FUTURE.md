# Future Request Intake Model

## Status

Future model. Not the current active build priority.

## Purpose

This file preserves the intended future Jira request intake model for organisation-wide digital requests.

The current active priority is existing Hopper consolidation, not public intake automation.

## Future Intake Principle

Jira may become the front door for the organisation to raise digital improvement, enhancement, support, dashboard, integration, and software requests.

A submitted request does not automatically become a governed Hopper initiative.

## Future Intake Flow

```text
Organisation raises Jira request
↓
Digital Team intake review
↓
Clarify / reject / duplicate / BAU / promote to Hopper
↓
If promoted, item enters Hopper priority screening
```

## Request Types

Recommended future request types:

- Digital Improvement Request
- Software Enhancement Request
- New Software / System Request
- Dashboard / Reporting Request
- Integration / Automation Request
- Digital Support Request

## Public Request Form Fields

Keep public forms simple.

Recommended public fields:

- Request title
- Request type
- Business area / department
- What problem or improvement are you raising?
- What happens today?
- What would you like to be different?
- Which system or process is affected, if known?
- How urgent is this?
- Who is the best person to discuss this with?
- Attach evidence, screenshots, or examples

## Internal Fields Not for Public Users

Do not ask normal users for:

- Safety & Duty Holder score
- Regulatory / Reputational score
- Financial / Commercial score
- Maturity track
- Roadmap track
- PEP route
- CTR impact
- DRB decision

These are Digital Team / DRB fields.

## Future Workflow

```text
New Request
↓
Intake Review
↓
Clarification Required / Duplicate / Reject / BAU / Promote to Hopper
```

## Current Parking Rule

Do not build this before the current Hopper consolidation model is operational.
