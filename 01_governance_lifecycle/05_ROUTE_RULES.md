# Route Rules

Approved Hopper items must be routed before the formal form is built.

Use only the approved route labels in `00_system_control/CONTROLLED_VOCABULARY.md`.

Implementation route: use where the main work is selecting, configuring, replacing, consolidating, or rolling out a system or platform.

Development route: use where the main work is building, changing, enhancing, integrating, automating, or reporting through an existing system.

> **Deprecated — "Hybrid route".** "Hybrid route" is not an approved route classification and must not be used or recorded as an active route. Where both delivery types appear to apply or the main delivery owner is unclear, classify the route as `TBC` and ask the Digital Lead before progressing. The controlled route-vocabulary split (separating Implementation Route and Support Route) is owned by the initiation-form-route-hardening workstream; this file does not redefine the route model.

Stage closeout and readiness wording are route-aware and governed by `00_system_control/07_GOVERNED_WORKFLOW_LOOPING_STANDARD.md`.

For each item, record:

- recommended route
- reason
- required supporting artefacts
- unanswered route questions
- confirmed owner or route lead

The route recommendation is not approval. It prepares the correct form and delivery handoff.
