# Create Process Mapping Pack

## Purpose

This coworker procedure converts a process matrix or workshop notes into a practical process mapping pack for governance, Blueworks, initiation form support, and developer/vendor scoping.

## Inputs

Accept:

- existing process matrix
- workshop notes
- screenshots or exported steps
- current-state notes
- future-state notes
- known bottlenecks
- system notes
- required enhancement notes

## Required Outputs

Produce the following pack:

1. Process Mapping Summary
2. Current-State Process Narrative
3. Future-State Process Narrative
4. Blueworks Build Brief
5. Bottleneck Register
6. Enhancement / Scope Register
7. Vendor / Developer Questions
8. Initiation Form Support Text
9. DRB Decision Points

## Output 1: Process Mapping Summary

Include:

- process reviewed
- reason for review
- main pain points
- main proposed changes
- likely route: implementation / development / hybrid
- governance implications

## Output 2: Current-State Process Narrative

Describe current flow in order.

Include roles, systems, handoffs, decisions, bottlenecks, and manual workarounds.

## Output 3: Future-State Process Narrative

Describe proposed future flow.

Include removed steps, changed steps, system changes, new controls, integrations, and competence/training needs.

## Output 4: Blueworks Build Brief

Provide a structured build brief that can be used to recreate or update the formal flow in Blueworks.

Use:

| Lane / Role | Step No. | Activity | System | Decision / Handoff | Issue | Future Change |
|---|---:|---|---|---|---|---|

## Output 5: Bottleneck Register

Use:

| Bottleneck | Current Impact | Root Cause | Affected Role / Team | System / Process Link | Recommended Change |
|---|---|---|---|---|---|

## Output 6: Enhancement / Scope Register

Use:

| Requirement | Route | In Scope? | Out of Scope? | Owner / Delivery Party | Acceptance Note | Open Question |
|---|---|---|---|---|---|---|

## Output 7: Vendor / Developer Questions

Questions should confirm capability, scope, assumptions, constraints, cost drivers, integration needs, data needs, and acceptance criteria.

## Output 8: Initiation Form Support Text

Provide paste-ready text for:

- current process issue
- proposed future process
- system impact
- scope boundary
- benefits
- risk if not addressed
- competence / training requirement

## Output 9: DRB Decision Points

List only the decisions needed from governance.

## Rule

Do not pretend the AI-generated process map is the controlled record. Blueworks remains the controlled process map environment.
